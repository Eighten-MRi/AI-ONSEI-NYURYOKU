# 音声認識精度向上プラン

## Context
現在 `gemini-2.0-flash-lite`（軽量モデル）を使っており、聞き取り精度に改善の余地がある。モデルのアップグレードと生成パラメータの最適化で、コード変更を最小限にしつつ精度を大幅に向上させる。

## 変更内容

### 1. モデルを `gemini-2.5-flash` にアップグレード（効果：大）
- **ファイル**: [main.py:41](main.py#L41)
- `MODEL_NAME = "gemini-2.0-flash-lite"` → `MODEL_NAME = "gemini-2.5-flash"`
- 音声理解・日本語認識が大幅に向上する最大のインパクト変更

### 2. `temperature=0` を設定（効果：中〜大）
- **ファイル**: [main.py:42](main.py#L42)
- 文字起こしは創造性不要。最も確率の高いトークンを確定的に選択させる
- 変更後:
```python
model = genai.GenerativeModel(
    MODEL_NAME,
    generation_config=genai.GenerationConfig(temperature=0)
)
```

### 3. システムプロンプトに精度重視の指示を追加（効果：中）
- **ファイル**: [main.py:61-88](main.py#L61-L88)
- プロンプト冒頭に「正確な聞き取り最優先」の指示を追加:
  - 聞き間違いや勝手な補完の禁止を明示
  - 聞こえた通りに忠実に書き起こすことを強調

## 変更しないもの
- **サンプルレート（16kHz）**: Gemini内部でリサンプリングされる可能性が高く、効果が不確実。変更しても3倍のデータ量になるだけの可能性あり
- **ノイズリダクション**: `noisereduce`等の追加は重い依存関係を生む。Geminiの上位モデルはノイズ処理も優秀なので、まずモデルアップグレードで様子を見る
- **依存関係**: 新しいライブラリの追加は不要

## 検証方法
1. アプリを起動し、右Altキーで録音テスト
2. 以下のパターンで精度を確認:
   - 通常の文章（短文・長文）
   - フィラー入りの発話（「えー」「あの」を含む）
   - 小声での発話
   - 「改行」「まる」「てん」コマンド
3. 以前うまく聞き取れなかったフレーズで改善を確認
